First article and Entry for r/dataisbeautiful DataViz Battle

January 20, 2018

Objectives

As first article on this R and dataviz dedicated blog, I present to you my entry for the DataViz battle of January 2018.

Rules

This month and the data source is a .htm webpage containing several algae growthrates according to light and temperature conditions.

How I work on this

For this project I used :

  1. R for
  • Data importatation
  • Data wrangling and cleaning
  • Final analysis
  1. Tableau Public
  • Data exploration
  • Detailed data dashboard

So let’s start with R !

algae <- algae[-3,] #remove duplicatedLight row

Quick analysis using data visualisation on Tableau

this dashboard was designed more for data exploration than knowledge presentation. The hypothesis I was able to make based on simply exploration through visualisation are :

Hypothesis

{{< alert info no-icon >}} 1. As only 9 algaes can grow at 5C and 15C at 10C on a total of 19 species, temperature seems to be the most important factor for algae growth. 2. More light is not directly linked to more growth, instead, it looks like a genus or specy preference. {{< /alert >}}

Facts

{{< alert success >}} 2. Saying that I cannot control Light and Temperature and these variables vary from their minimum and maximum during the year, the best growing algae is Chlorella ellipsoidea as its mean growthrate is the highest. {{< /alert >}}

Hypothesis testing

We are back in R in order to model the temperature effect on algae growth for any Light condition.

#lets plot that
box1 <- ggplot(algae, aes(fct_inorder(Temperature), GrowthRate))+
geom_boxplot(aes(group=Temperature), outlier.alpha = 0.4)

box2 <- ggplot(algae, aes((Light), GrowthRate))+
geom_boxplot(aes(group=Light), outlier.alpha = 0.4)+theme(axis.title.y = element_blank())

box1 + box2

We see a global tendency of Growthrate increasing as Temperature increases. As for Light, nothing clear emerge from this representation.

In order to assess and quantify the effect of Temperature and light on algae growth, we fit an anova model :

fitanova <- aov(algae$GrowthRate~algae$Temperature*algae$Light+algae$Light*algae$Temperature)
summary(fitanova)

The results of this modelisation tell us that only temperature has significant effet on growth (or that we don’t have enough evidence to show Light effects if you prefer).

If we consider Temperature effect as continuous and build a linear model, we can provide a rough effect quantification on growth :

fitlm <- lm(algae$GrowthRate~algae$Temperature)

ggplot(algae, aes(as.numeric(levels(Temperature)[Temperature]), GrowthRate))+geom_point(aes(color=Genus))+geom_smooth(method="lm")+
stat_summary(fun.y = mean, geom="line", aes(group=Genus, color=Genus))+labs(title =paste("Growthrate =",signif(fitlm$coef[[2]], 3),"* Temperature +",signif(fitlm$coef[[1]],3 )), subtitle=paste("Adj R2 = ",signif(summary(fitlm)$adj.r.squared, 5), " p-value =",signif(summary(fitlm)$coef[2,4], 5)))+theme(plot.title = element_text(hjust=0.5), plot.subtitle = element_text(hjust=0.5))

{{< alert warning >}} I am not rigorously sure about this assumption of effect quantification, so if i’m wrong please, contact me on twitter {{< /alert >}}

At the begining of this project, I tried to use only R to build the DataViz I wanted to make, but this quickly turned in a very complicated task…

algae_means <- algae %>% group_by(Specy,Light) %>% mutate(growth_minus_Temp =mean(GrowthRate, na.rm=T)) %>% ungroup() %>%  group_by(Specy,Temperature) %>% mutate(growth_minus_Light =mean(GrowthRate, na.rm=T)) %>% ungroup() %>% group_by(Specy) %>% mutate(total_mean=mean(GrowthRate, na.rm=T))


filter(algae_means, is.na(total_mean))
library("packcircles")
library("viridis")
library("ggiraph")
pack <- circleProgressiveLayout(unique(algae_means$total_mean),sizetype = "area")
pack <- pack[complete.cases(pack),]



data <-  bind_cols(algae_means %>% select(Specy, Genus, total_mean) %>% filter(total_mean>=0)%>% unique(), pack)

pack <- cbind(Specy=data$Specy, pack)
pack <- pack %>% select(x,y,radius,Specy)
pack$radius <-  rnorm(2,mean = 0.95,sd = 0.05)*pack$radius

data$text <- paste("Genus : ", data$Genus,"\n","Specy: ",data$Specy, "\n", "Mean Growthrate : ",data$total_mean)

dat.gg <- circleLayoutVertices(pack,npoints=60,idcol = "Specy")
dat.gg <- dat.gg %>% mutate(Genus=word(dat.gg$id,1))

p <- ggplot() + 
  geom_polygon_interactive(data = dat.gg, aes(x, y, group=id, fill=id, data_id=id, tooltip=data$text[id]), colour = "black", alpha = 0.75) +
  scale_fill_brewer(type="qual", palette=3) +
  geom_text(data = data, aes(x, y, label =Specy, size=(2*radius)), color="black") +
  theme_void() + 
  theme(legend.position="none", plot.margin=unit(c(0,0,0,0),"cm") ) + 
  coord_equal()+
  labs(title = "Mean Growth Rate of algae by species", caption="Colored by Genus. Mean Growth is idependant of : \n   - Light (between 2,5-5K Lux) \n  - Temperature (between 5-25 Celsius)")+
  theme(plot.title = element_text(size=24, hjust=0.5,face = "bold"), plot.caption = element_text(size=14, hjust=0,face = "italic"))
 
widg=ggiraph(ggobj = p, width_svg = 7, height_svg = 9)
 
widg

Although this chart is OK, it is far from finished, and I still spent about 2 hours to get it ! This is why I think combining R and Tableau is good for knowledge discovery process. R for data wrangling, tidying and summary ; Tableau for quick visualisations and experimentations on data. Then I usually come back to R to build a more precise and customized visualisation when Tableau doesn’t fill my needs.
